# JSON Schema Notes

GreenLeaf Advisor uses structured JSON outputs generated by Python.

The JSON files act as verified input for the advisor layer. The advisor should only use values from these files and should not invent unsupported numbers.

## plot_priority_summary.json

Purpose:
Ranks plots that need attention first.

Main fields:
- summary_type
- method
- top_priority_plots

Each priority plot includes:
- plot_id
- urgency_score
- total_alerts
- avg_plant_stress
- avg_response_delay_days
- no_response_count
- reason

## response_timing_summary.json

Purpose:
Summarizes response timing and stress recovery patterns.

Main fields:
- summary_type
- important_note
- delay_bucket_summary
- alert_type_summary

Delay bucket records include:
- delay_bucket
- alert_count
- avg_stress_recovery
- avg_response_delay_days
- same_day_response_rate
- worsening_stress_count

Alert type records include:
- alert_type
- alert_count
- avg_stress_recovery
- avg_response_delay_days
- worsening_stress_count

## precision_roi_summary.json

Purpose:
Summarizes precision action cost and season ROI value.

Main fields:
- summary_type
- precision_cost
- routine_cost
- precision_action_count
- routine_action_count
- precision_cost_share
- total_precision_benefit
- avg_season_roi
- total_season_profit
- avg_yield_kg_m2
- avg_marketable_ratio
- input_type_cost_summary

## financing_memo_summary.json

Purpose:
Provides lender-ready season outcome metrics.

Main fields:
- summary_type
- avg_season_roi
- total_profit_cad
- total_revenue_cad
- total_cost_cad
- total_precision_benefit_cad
- avg_yield_kg_m2
- avg_marketable_ratio
- top_roi_plots
- top_precision_benefit_plots

## Advisor Output Requirements

The advisor should return:

{
  "summary_type": "",
  "key_findings": [],
  "recommendations": [],
  "data_limitations": [],
  "farmer_summary": "",
  "lender_summary": ""
}

## Guardrails

The advisor must:
- use only verified JSON values
- avoid unsupported claims
- avoid inventing numbers
- label response timing as association, not proven causality
- explain recommendations using KPI evidence
